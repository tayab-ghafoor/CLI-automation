## ✨ Key Features

1. **Email-Based Authentication**
   - Users login with email and password
   - Automatic session management

2. **Secure Password Storage**
   - Passwords hashed with SHA256
   - Never stored in plaintext

3. **Session Management**
   - 8-hour session duration
   - Automatic expiration after 8 hours
   - One active session per user

4. **User Profiles**
   - View email, name, creation date, last login
   - Change password anytime

5. **Admin Capabilities**
   - First registered user is admin
   - Can delete user accounts

6. **Error Handling**
   - Validates email format
   - Checks password strength
   - Prevents duplicate accounts
   - Handles expired sessions gracefully

## 💡 Important Notes

1. **First User is Admin:** The first person to register becomes the admin
2. **8-Hour Sessions:** After 8 hours, users need to login again
3. **No Password Reset:** Currently, users must have admin delete and re-register
4. **Data Stored Locally:** All user data is stored in `data/` folder
5. **Email-Based:** Uses email as unique identifier

## 🆘 Troubleshooting

| Issue | Solution |
|-------|----------|
| "Email already registered" | Use different email or login |
| "Password too weak" | Use 8+ chars with uppercase & digit |
| "Invalid email format" | Use proper email format |
| "Session expired" | Login again |
| "User not found" | Check email spelling |
